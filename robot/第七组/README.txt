第七组的树莓派开发手册
1：树莓派上Python的pip使用
如何安装pip网上有很多教程这里就不再阐述，关键是在树莓派上直接使用pip指令会无法下载相应的包(电脑上不影响)，我们需要换用国内镜像源，有两种方式，一种为修改配置文件，同样有许多教程，主要介绍我常用的一种，使用 pip install -i 镜像源网址 包名的指令，如我们使用国内清华镜像安装numpy：在命令行中输入pip install -i https://pypi.tuna.tsinghua.edu.cn/simple numpy即可，注意尽量使用pip而不是pip3（言外之意使用python2而非python3），否则包可能安装到python3下
2：Snowboy与pyaudio的注意事项
Snowboy是一个非常好用的唤醒词，网上找到的代码多为非循环方式，为了提高交互性应该改为循环方式，方法为函数的无穷递归调用，在回调函数中加入os.sys(‘命令行命令’)即可调用其他函数，非常注意如果看到网上在你pyaudio出问题时推荐装pulseaudio包千万别装，会重定向你的默认音频输入输出设备，与其他语音包存在很大冲突，你可以自己在pi目录下创建.asoundrc文件，并修改配置（可在网上找到相应的配置代码），这样就可以解决输入输出设备问题（如果还没有声音可以检查一下树莓派右上角的小喇叭是否设置成了HDMI，是的话得修改为analog）
3：opencv实现人脸识别的注意事项
树莓派的性能并不好，使用opencv进行人脸识别效率会非常差，视频帧数非常低，为了改善这样的情况，可以使用python的multiprocessing模块，使用后视频帧数能够到达5到6帧，网上也有这方面的详细资料这里不做介绍。
4：一些关于进程通信的小技巧
一般来说涉及到语音图像结合的内容都会用到进程间的通信，可以使用的有os.sys，或者subprocess等库，需要自己网上去查看资料，了解使用方法，但也可以使用一些巧妙地方法来避免使用进程，即文件操作，比如说语音检测到有新的身份录入请求，则创建相应文件，与此同时，人脸检测的程序每隔一定的帧数就检测相应的文件夹，一旦发现文件数量变了，就立刻更新训练样本，重新训练模型，这样就能实现实时的身份生成。利用这样的技巧不需要去等待系统切换进程，节约了系统开销和时间，推荐使用。
5：人脸追踪的实现方式
人脸追踪其实实现起来非常简单，首先我们得确定整个图像的中心点的坐标，还有人脸的中心坐标，这两点看懂opencv的代码即可自己计算得到，然后我们需要以图像中心点为中心划分出一个稳定区，区域的大小和形状可以自行调节，然后对于其他区域，根据人脸的中心坐标相对与图像中心相对方位，利用串口通信给stm32发送信号，32接受信号进行相应的处理即可完成
6：小车的智能跟随
智能跟随主要可以考虑两种方案，一种是完全交给32做，利用超声波传感器完成功能，还有一种难度比较高的是利用树莓派的图像来做，前者只能做到前进和停止的跟随，后者利用对象识别（识别人）可以做到前后左右和停止的跟随，如利用后者需要考虑树莓派的性能问题，建议另外在申请一块树莓派单独做图像部分，实现原理与人脸追踪相仿，不再阐述
